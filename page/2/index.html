需求场景

由于我的Linux版本是i386的，无法愉快的使用云笔记之类的应用。我平时办公使用的是Mac，回家之后家里是使用老旧电脑配置的Linux系统，不同的电脑，不同的系统之间，很需要文件的同步和笔记应用的同步。这样不至于使每个电脑成为信息孤岛。这个问题困扰了我很久。

其实家里主力电脑是用来上网课，浏览资料，也做做资料。Linux本身无法愉快的使用微信和QQ，这点倒是不在乎，这样在学习时才能完全实现浸入式。但是与平时上班时使用的MacbookPro要做一定同步，才能保持工作与生活的连贯性。特别是平时使用Mac和iphone已经能够做到很好的互联，使用的工具是icloud和onedrive，另外还有“有道云笔记”这样的应用。

为解决github访问速度慢而遇到问题

2020-03-20

一直以来，在使用github这个问题上，相信跟许多朋友一样都会遇到速度慢的问题。由此也一直在网上找些相关的解决方法，可是都不那么尽人意，网上的解决方法基本是直接修改hosts文件。我来试了试，却不那么尽人意。

网上的方法大致这样。

1. 修改本地hosts文件

我在mac终端上，sudo -i进入root，vim打开/etc/hosts文件。

2. 增加http://github.global.ssl.fastly.net 和 http://github.com 的映射

相信每个朋友通过ipaddress.com查询出来的都略微有些不同。

3. 在终端输入ping github.com能通了

192:etc root# ping github.com
PING github.com (140.82.113.3): 56 data bytes
64 bytes from 140.82.113.3: icmp_seq=0 ttl=46 time=294.942 ms
64 bytes from 140.82.113.3: icmp_seq=1 ttl=45 time=358.485 ms
64 bytes from 140.82.113.3: icmp_seq=2 ttl=45 time=248.815 ms
64 bytes from 140.82.113.3: icmp_seq=3 ttl=45 time=251.610 ms
64 bytes from 140.82.113.3: icmp_seq=4 ttl=44 time=292.832 ms
64 bytes from 140.82.113.3: icmp_seq=5 ttl=45 time=256.472 ms

可是网页去一直打不开。

4. 修改

重新回到/etc/hosts，将140.82.113.3 github.com这句删除，保存并退出。问题就解决了。

在我一知半解的情况下，觉得应该是https协议导致能ping通，却打不开网站。不知道是否为正解。

5. TODO

待理清原因，或发现更好的解决方法再补充。
